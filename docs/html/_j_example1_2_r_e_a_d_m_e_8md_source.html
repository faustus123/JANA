17 This example demonstates how to read events into JANA. No actual processing
18 of events is done. They are simply created, placed in the main "Physics Events"
19 queue, and then pulled from the queue and discarded. A number of features
20 that would be exercised in a standard JANA implementation are not used here
21 for simplicity. See other examples for those.
22 
23 Details
24 --------------
25 JANA generates events by calling the GetEvent() method of a JEventSource object.
26 The job of the JEventSource is to read in an "event" so it can be placed on a
27 queue for further processing. Here, "event" may be a complete physics event
28 or may be some large buffer containing a block of many physics events. In
29 general, the event source should only read the data with minimal processing
30 on it. This is because the GetEvent() calls are serialized so only one thread tries
31 to read from the source at a time. It is worth noting here that if multiple sources
32 are given to a single JANA job, it may read from those in parallel, but each one
33 will be read by only one thread at a time.
34 
35 For simple implementations like this however, one may just want to unmarshall
36 the data in GetEvent(), placing the items in members of the JEvent subclass.
37 
38 The JEventSourceGenerator class is what allows JANA to simultaneously support
39 reading files from multiple types of sources. Each source that is specified for a
40 job is passed as a string to the CheckOpenable() method of all
41 JEventSourceGenerator objects registered in the application. Each examines
42 the string and returns a value from 0-1.0 indicating how confident it is
43 that it can read events from this source. For example, if you had two
44 file formats that you wanted to read events from, their corresponding
45 JEventSourceGenerator classes could simply check the suffix of the source
46 and return 0 for all but the one type it can read. JANA will call the
47 MakeJEventSource method of the generator chosen to open the source.
48 In this example, the CheckOpenable() method always returns 0.5. It
49 could return anything >0 since it is the only source available in this
50 example and so will always be chosen.
51 
52 The source for this example is compact enough that it is contained in a single
53 file: JExample1.cc. It defines 3 classes and the special InitPlugin()
54 routine that gets called when the plugin is attached. See comments in
55 the source for more details.
