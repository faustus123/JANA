52 /// This templated class is used to generate JEventSource based objects
53 /// to handle reading events into JANA. Multiple JEventSourceGenerator
54 /// objects may exist in a given program, each representing a different
55 /// input type. For example, one source may be used to read raw data
56 /// while another may be used to read DST formats. A third may be used
57 /// to read from a network connection for online applications. For each
58 /// source specified in a given job, JANA will determine which JEventSource
59 /// class to use to read from it. It does this via the CheckOpenable method
60 /// of JEventSourceSourceGenerator.
61 ///
62 /// A new instance of the JEventSource subclass will be created for each
63 /// input source in the job. For example, if two files of the same type,
64 /// are specified then the JEventSourceGenerator corresponding to that
65 /// file type will be used to create two JEventSource objects.
66 ///
67 /// REQUIREMENTS OF JEventSource SUBCLASS:
68 /// --------------------------------------
69 /// - constructor that accepts (std::string, JApplication*)
70 /// - static std::string GetDescription(void) method
71 ///
72 ///
73 /// NOTE ON CheckOpenable:
74 /// --------------------------------------
75 /// This method should "peek" at the source to see if it is one that
76 /// it can open. It should return a value between 0 and 1 inclusive
77 /// with 0 meaning "cannot open" and 1 meaning "absolutely can open".
78 /// The check can be as simple as looking at the source name (e.g.
79 /// does it have a specific suffix) or more involved (e.g. opening
80 /// the file and checking for a magic header). Note that it should
81 /// not be assumed that the source represents a file name. It could
82 /// indicate a URL or how to connect to some other non-file source.
83 ///
84
